Spin

A powerful development environment manager that helps you run and manage multiple processes and services in your development workflow.

Get Started

Installation

Prerequisites

  • tmux - Required for process management
    Used to manage and isolate development processes

Quick Install

Install Spin:

Download from GitHub Releases

Verify installation:

spin

Platform-Specific Setup

macOS

Install prerequisites using Homebrew:

brew install tmux
Linux

Install prerequisites using apt (Ubuntu/Debian):

sudo apt update && sudo apt install tmux

Or using yum (RHEL/CentOS):

sudo yum install tmux

Commands

spin up [app-name]

Start the development environment and required services for an application. Automatically manages services defined in your configuration.

spin up           # Start the app and required services
spin up myapp     # Start the app in the myapp directory

spin down

Stop all running processes, clean up the development environment, and stop any services that were started.

spin down         # Stop all processes and services

spin ps

List all running processes and their status.

spin ps           # Show process list

spin logs [process-name]

View the output logs for a specific process.

spin logs web     # View web process logs

spin debug [process-name]

Attach to a process in debug mode (useful for interactive debugging sessions).

spin debug web    # Debug web process

spin setup [app-name]

Initialize a new application with Spin configuration.

spin setup myapp              # Setup new app in myapp directory
spin setup myapp --repo=org/name  # Setup with specific repository
spin setup . --force         # Setup in current directory

spin config

Manage Spin configuration settings.

spin config show              # Show current configuration
spin config set-org myorg     # Set default organization

Configuration

Configuration is read from the spin.config.json file in your project directory.

Configuration File

The main configuration file for your application. Here's an example of a Rails application configuration with service dependencies:

{
  "name": "test_rails",
  "version": "1.0.0",
  "type": "rails",
  "repository": {
    "organization": "afomera",
    "name": "test_rails"
  },
  "dependencies": {
    "services": ["postgresql", "redis"],  // Supported services: postgresql, redis, sqlite3
    "tools": ["ruby", "bundler"]
  },
  "scripts": {
    "setup": "bundle install && rails db:setup",
    "start": "rails server",
    "test": "rails test"
  },
  "env": {
    "development": {}
  },
  "rails": {
    "ruby": {
      "version": "3.4.1"
    },
    "database": {
      "type": "postgresql",
      "settings": {
        "database": "myapp_development"
      }
    },
    "rails": {
      "version": "8.0.1"
    },
    "services": {
      "redis": true,    // Enable Redis for Action Cable/Caching
      "sidekiq": true   // Enable Redis for Sidekiq
    }
  }
}

Service Management

Spin provides automatic service management for your development environment. Services are defined in your configuration file under the dependencies.services section.

Supported Services

PostgreSQL

Managed PostgreSQL database service.

{
  "dependencies": {
    "services": ["postgresql"]
  },
  "rails": {
    "database": {
      "type": "postgresql"
    }
  }
}
Redis

In-memory data store for Action Cable, caching, and Sidekiq.

{
  "dependencies": {
    "services": ["redis"]
  },
  "rails": {
    "services": {
      "redis": true,
      "sidekiq": true
    }
  }
}
SQLite3

Lightweight file-based database option.

{
  "dependencies": {
    "services": ["sqlite3"]
  },
  "rails": {
    "database": {
      "type": "sqlite3"
    }
  }
}

Service Lifecycle

When you run spin up, Spin will:

  • Read service requirements from your configuration
  • Check if each required service is already running
  • Start any required services that aren't running

When you run spin down, Spin will:

  • Stop all services that were started by Spin
  • Clean up any temporary service data

Procfile.dev

Define additional processes to run alongside your main application:

web: bundle exec rails server -p 3000
worker: bundle exec sidekiq
css: yarn build:css --watch
js: yarn build --watch

Development Workflow

  1. Create a spin.config.json in your project
  2. Configure required services in the dependencies section
  3. Add a Procfile.dev if you need multiple processes
  4. Run spin up to start your development environment and services
  5. Use spin ps to monitor running processes
  6. View logs with spin logs [process]
  7. Debug with spin debug [process] when needed
  8. Stop everything with spin down
